## 🛠️ 상세 수정 내역

### 1. JWT 및 인증 시스템 고도화 (`api.ts`, `App.tsx`, `StompContext.tsx`)
- **토큰 사전 검증(Pre-emptive Guard)**: API 요청 및 웹소켓 연결 전 JWT의 `exp` 필드를 검사하여 만료된 요청을 클라이언트 단에서 원천 차단.
- **인증 에러 통합 핸들링**: 
    - 401 에러 시 중복 알림을 억제하고 단 하나의 '세션 만료' 안내만 제공.
    - 세션 만료 시 즉시 홈(`/`)으로 이동 및 로그인 모달 자동 팝업 로직 구현.
- **STOMP 안정화**: 인증 실패 시 5초 간격의 무한 재연결 루프를 즉시 중단(`deactivate`)하도록 보강.

### 2. 관리자 페이지 기능 및 레이아웃 최적화 (`AdminMemberPage.tsx`)
- **API 규격 최신화**: 상세 조회 엔드포인트를 패스 베리어블 방식(`/api/boardMaster/readBoardMasterDetail/{code}`)으로 전환.
- **데이터 정렬 및 바인딩**: 
    - 게시판 및 상세 코드 목록에 `sortOrder` 기준 정렬 로직 적용.
    - 비동기 데이터 로드 시 폼 데이터가 즉시 반영되도록 `useEffect` 동기화 로직 추가.
- **디자인 복구 및 정적화**: 
    - 임의 변경된 `table` 구조를 원래의 프로젝트 표준 `div` 기반 레이아웃으로 완벽 복구.
    - 탭 전환 시 시각적 잔상을 주는 모든 애니메이션과 `pulse` 효과를 제거하여 즉각적인 화면 전환 구현.
- **그룹코드 필드**: `BoardFormModal`의 그룹코드 필드를 텍스트 입력형태로 바꾸고 읽기 전용(`disabled`) 처리.

### 3. 실시간 알림 시스템 정상화 (`NotificationContext.tsx`)
- **구독 주소 복구**: 유저 고유 식별자가 포함된 개인 알림 채널(`/sub/user/{memberId}`)로 주소 보정.
- **유연한 파싱 도입**: 서버 응답의 다양한 필드명(content, message, notiContent 등)을 `content` 하나로 통합 매핑하여 데이터 유실 방지.

### 4. 주요 페이지 런타임 에러 해결
- **ExamDetailPage**: 작성자 이름(`authorName`) 누락 시 발생하던 크래시 해결 및 파일 다운로드 로직 보정.
- **WritePostPage**: 누락되었던 `handleCancelClick`, `handleDiscard` 함수 복구 및 모달 스크롤 방지 적용.
- **ProfileDropdown**: 프로필 업로드 시 API 언래핑 데이터 처리 오류 해결.
- **공통**: 모든 모달 오픈 시 배경 화면 스크롤 잠금(`overflow: hidden`) 로직 일괄 적용.
