## ✦ 핵심 작업 목표

- 관리자 메뉴 내 공통 상세 코드 조회 API 규격 최신화 및 정렬 로직 도입

## 🛠️ 상세 수정 내역

### 1. JWT 무한 로그 차단 시스템 구축 (`api.ts`, `StompContext.tsx`)
- **토큰 사전 검증(Pre-emptive Guard) 도입**: 
    - 순수 JS 기반의 `isTokenExpired` 헬퍼 함수 구현.
    - API 요청 및 STOMP 연결/구독/발행 전 토큰의 `exp` 필드를 검사하여 만료된 경우 서버 요청을 원천 차단.
- **웹소켓 강제 중단 로직 강화**:
    - **STOMP 에러 핸들링**: `onStompError`에서 401 또는 Expired 관련 메시지 수신 시, 즉시 `client.deactivate()`를 호출하여 5초 간격의 무한 재연결 루프를 물리적으로 파괴.
    - **전역 이벤트 동기화**: `api.ts`에서 발생하는 `auth-error` 이벤트를 STOMP 컨텍스트가 리스닝하여 인증 실패 시 즉시 모든 소켓 인스턴스를 제거.
- **기대효과**: 서버 리소스 낭비 방지 및 불필요한 에러 로그 누적 차단.

### 2. 관리자 공통 코드 관리 기능 최적화 (`AdminMemberPage.tsx`)
- **API 엔드포인트 최신화**: 상세 코드 조회 API를 기존 `getCommonDetailCode`에서 `/api/common-code-group-detail/readDetailCommonDetailCode/`로 변경 (백엔드 최신 규격 반영).
- **데이터 정렬 로직 도입**: 서버에서 받아온 상세 코드 리스트를 프론트엔드에서 `sortOrder` 숫자를 기준으로 오름차순 정렬하여 화면에 표시.
- **게시판 순서 제어 보강**: 게시판 설정 목록에 `sortOrder`를 표시하고, 수정 모달에 입력 필드를 추가하여 노출 순서 관리 기능 완성.
- **도구 오류 복구**: 파일 수정 중 발생한 에러를 `generalist` 서브에이전트 협업 및 `write_file`을 통해 안정적으로 복구 및 반영.

### 3. 사용자 경험(UX) 개선: 모달 배경 스크롤 방지
- **스크롤 간섭(Scroll Bleeding) 해결**: 모든 모달 컴포넌트(`Login`, `SignUp`, `Confirm`, `Admin` 등)가 열릴 때 배경 화면이 스크롤되는 현상을 차단.
- **로직 표준화**: `useEffect`를 활용하여 모달 활성 시 `body { overflow: hidden }`을 적용하고, 닫힐 때 자동으로 원복하도록 설계.
- **적용 대상**: 공통 모달 6종 및 관리자 설정 전용 내부 모달 3종 전체 적용 완료.
