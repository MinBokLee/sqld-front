# 📅 작업 일지 (2026-04-08) - WebSocket 통합 리팩토링 및 실시간 오픈채팅 완성

## 🎯 주요 목표 (Goals)
- 중복 연결 문제를 해결하기 위한 **WebSocket 단일 소켓 공유 구조(StompContext)** 리팩토링.
- 실시간 오픈채팅의 입장/퇴장 로직 고도화 및 백엔드 신규 규격(`QUIT`, `notiId`, `senderName`) 연동.
- 모바일 및 반응형 환경에서의 **UI 겹침(Overlapping) 및 Z-Index** 최적화.

---

## 🛠️ 상세 작업 내역

### 1. WebSocket 통신 구조 전면 개편 (Architecture Refactoring)
- **StompContext 신설**: 최상위에 통신 본부를 두어 브라우저 탭당 **단 하나의 WebSocket 연결**만 맺도록 구조 통합.
- **리소스 공유**: `NotificationContext`와 `ChatContext`가 별도의 소켓을 생성하지 않고 `StompContext`의 클라이언트를 공유하여 **서버 부하 50% 절감**.
- **안정성 강화**: React Strict Mode 및 리렌더링 시 발생하던 이중 연결 버그를 구조적으로 원천 차단.

### 2. 실시간 오픈채팅 (Open Chat) 기능 고도화
- **백엔드 규격 완벽 대응**:
    - `senderName` 필드 적용으로 별도 조회 없이 닉네임 즉시 표시.
    - `QUIT` 타입 수용: 사용자가 브라우저를 닫아도 서버 감지를 통해 "퇴장하셨습니다" 문구가 타인에게 단 한 번 정상 전파되도록 구현.
- **UX 정밀 보정**:
    - **본인 입장 메시지 필터링**: 나에게는 입장 안내를 숨기고 타인에게만 나의 입장을 알리는 세련된 UX 적용.
    - **시스템 메시지 UI**: `ENTER`, `QUIT` 메시지를 중앙 캡슐 스타일로 분리하여 대화와 시각적 구분.
    - **자동 스크롤**: 새 메시지 수신 및 창 오픈 시 최하단 고정 로직 강화.

### 3. 알림 시스템(Notifications) 안정화
- **실시간 읽음 처리**: 백엔드에서 생성된 `notiId`를 프론트엔드 `id` 필드에 매핑하여 클릭 시 즉각적인 DB 읽음 상태 반영 완성.
- **라우팅 자동 변환**: 백엔드의 `/board/view?boardId=...` 형식을 프론트엔드 라우트인 `/exam/...`으로 변환하는 `normalizeUrl` 로직을 통해 이동 경로 정상화.

### 4. 반응형 UI 및 웹 표준 최적화
- **Z-Index 계층 정립**: 채팅창(`z-40`), 헤더 드롭다운(`z-70`), 모달(`z-100+`)로 우선순위를 재배치하여 UI 가려짐 현상 해결.
- **모바일 대응**: 작은 화면에서 채팅창 크기와 여백을 유동적으로 조절(`calc(100vw-32px)`)하여 간섭 최소화.
- **환경 설정**: Vite 환경에서의 `global is not defined` 오류를 위한 폴리필 설정 추가.

## ⚠️ 향후 과제
- **대화 내역 관리**: 메시지 누적 시 성능 저하 방지를 위한 가상 리스트(Virtual List) 도입 검토.
- **상태 보존**: 브라우저 새로고침 시에도 이전 대화 맥락이 유실되지 않도록 로컬 캐싱 전략 수립 예정.
