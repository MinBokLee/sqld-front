# 작업 요약 (Work Summary) - 2026-05-01

## 1. 운영 환경 설정 및 빌드 최적화
- **package.json 수정:**
  - 빌드 스크립트의 모드 이름을 `prod`에서 표준인 `production`으로 통일.
  - 이를 통해 빌드 시 `.env.production` 파일의 환경 변수(API BASE URL 등)가 정상적으로 로드되지 않던 문제 해결.
- **검증:** `npm run build` 테스트를 통해 운영 환경 변수 로딩 확인 완료.

## 2. 공통코드 및 게시판 정렬 시스템 개선
- **입력 버그 수정:**
  - `AdminMemberPage.tsx`에서 정렬 순서(`sortOrder`) 입력 시 '0'이나 기존 값이 지워지지 않던 로직 개선.
  - 빈 문자열 입력을 허용하여 사용자가 값을 지우고 새로 입력할 수 있도록 수정.
- **범위 검증(Validation) 추가:**
  - 항목(게시판, 그룹, 코드)의 현재 총 개수를 기준으로 입력 가능 범위를 1부터 Max까지 제한.
  - 범위를 벗어난 입력 시 "정렬 순서는 1부터 [N] 사이여야 합니다. ⚠️" 토스트 알림 발생 및 제출 차단.
- **UI 가이드 강화:**
  - 각 설정 모달의 입력 필드 우측 상단에 `Max: N` 표시를 추가하여 직관적인 입력 가이드 제공.

## 3. 인증 시스템 고도화 (JWT Silent Refresh)
- **자동 토큰 갱신 구현:**
  - Axios 응답 인터셉터에서 `401 Unauthorized` 감지 시 즉시 `/api/sign/reissue` 호출.
  - 갱신 성공 시 실패했던 원래 요청을 새 토큰으로 자동 재시도하여 무중단 사용자 경험 구현.
- **WebSocket(STOMP) 연동 개선:**
  - 소켓 연결(`connect`) 시도 직전 토큰 만료 여부를 사전 체크.
  - 만료된 경우 토큰 갱신 로직을 먼저 실행한 후 갱신된 토큰으로 안전하게 연결하도록 보강.
- **보안 및 상태 동기화:**
  - `UserContext`에 `refreshToken` 관리 로직 통합.
  - 토큰 갱신 시 앱 전역에 이벤트를 발송하여 유저 상태를 실시간으로 동기화.
  - 재귀 호출 방지를 위해 갱신 요청에는 순수 Axios 인스턴스 사용.

## 4. 검색 알림 방식 개선 (UX 최적화)
- **모달에서 토스트로 전환:**
  - 헤더 검색창 및 게시판 내 검색창에서 검색어 미입력 시 발생하던 차단형 모달(`showAlert`)을 비차단형 토스트(`showToast`)로 변경.
  - 사용자의 작업 흐름을 방해하지 않으면서 필요한 경고 메시지를 전달하도록 개선.

## 5. 게시판 첨부파일 관리 시스템 고도화
- **실시간 미리보기 UI 구현:**

- **업로드 전 파일 관리 기능:**
  - 게시글을 등록하기 전, 잘못 올린 파일을 즉시 제거할 수 있는 삭제 버튼 추가.
  - 글 수정 시에도 기존 첨부파일을 확인하고 선택적으로 삭제할 수 있는 로직 완비.
