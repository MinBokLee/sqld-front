# 📅 작업 일지 (2026-04-06) - 실시간 알림 및 오픈채팅 시스템 구축

## 🎯 주요 목표 (Goals)
- Redis 및 WebSocket을 활용한 **실시간 알림(댓글/대댓글)** 기능 구현.
- 사용자가 실시간으로 대화할 수 있는 **오픈채팅(Open Chat)** 기반 마련.
- Vite 환경에서의 WebSocket 통신 안정화 및 각종 런타임 오류 정규화.

---

## 🛠️ 상세 작업 내역

### 1. 실시간 알림 시스템 (Real-time Notifications) 완성
- **기술 스택**: `SockJS`, `@stomp/stompjs` 활용.
- **구현 내용**:
    - `NotificationContext` 구축: 로그인 시 `/sub/user/{memberId}` 자동 구독.
    - **댓글/대댓글 알림**: 백엔드 로직에 맞춰 수신된 알림을 `framer-motion` 토스트로 즉시 노출.
    - **읽음 처리 연동**: 백엔드 가이드(`notiId` 필드)를 적용하여 알림 클릭 시 `PUT /api/notification/read/{notiId}` 호출.
    - **라우팅 자동 변환**: 백엔드의 `/board/view?boardId=...` 형식을 프론트엔드 라우트인 `/exam/...`으로 변환하는 `normalizeUrl` 로직 적용.
    - **마이페이지 확장**: '알림 내역' 탭을 신설하여 과거 알림 목록 조회 및 관리 기능 추가.

### 2. 실시간 오픈채팅 (Open Chat) 기능 구축
- **상태 관리**: `ChatContext`를 신설하여 메시지 송수신 및 대화 히스토리 관리.
- **채팅 UI**: 화면 우측 하단에 위치한 플로팅 채팅창(`OpenChat.tsx`) 개발.
- **기능 세부 사항**:
    - `ENTER` 타입 메시지: 사용자 접속 시 "OOO님이 입장하셨습니다" 안내 문구 자동 전파 (백엔드 브로드캐스트 연동).
    - `TALK` 타입 메시지: 닉네임과 말풍선이 조합된 세련된 대화 UI 구현.
    - **자동 스크롤**: 새 메시지 수신 시 또는 채팅창 오픈 시 최하단으로 자동 스크롤.
    - **히스토리 로드**: 입장 시 `GET /api/chat/getChatHistory/OPEN_CHAT`을 통해 최근 대화 50개 동기화.

### 3. 환경 최적화 및 안정화 (Bug Fixes)
- **Vite 설정**: WebSocket 통신을 위해 `vite.config.ts`에 `/ws-stomp` 프록시(`ws: true`) 및 `global` 폴리필 추가.
- **문법 에러 수정**: `Header.tsx`의 JSX 중첩 오류 및 `MyPage.tsx`의 `Bell` 아이콘 임포트 누락 해결.
- **데이터 정규화**: 백엔드 필드명 변경(`notiId`, `senderName` 등)에 유연하게 대응하도록 인터페이스 및 매핑 로직 강화.

## ⚠️ 향후 과제 및 특이 사항
- **백엔드 정책 준수**: `ENTER` 메시지는 서버에서 생성한 문구를 우선적으로 사용하도록 로직 수정 완료.
- **성능 고려**: 채팅 메시지 누적에 따른 브라우저 부하 방지를 위한 최적화 검토 필요.
- **사용자 경험**: 채팅창을 다시 열었을 때 스크롤 위치 보존 및 데이터 무결성 재검증 예정.
