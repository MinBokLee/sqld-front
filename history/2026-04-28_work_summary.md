## ✦ 핵심 작업 목표
- 오픈채팅 내 실시간 접속 유저 명단 및 인원수 동기화 기능 구현
- 백엔드 실제 응답 구조(`res.result.data`) 분석 및 연동 최적화
- 사용자 경험(UX) 개선을 위한 명단 토글 UI 도입

## 🛠️ 상세 수정 내역

### 1. 실시간 데이터 스트리밍 고도화 (`ChatContext.tsx`)
- **이중 구독(Multi-Subscription) 체계 구축**: 기존 메시지 채널 외에 `/sub/chat/room/OPEN_CHAT/presence` 채널을 동시에 구독.
- **히스토리 기능 비활성화**: 초기 진입 속도 향상 및 실시간성 집중을 위해 `fetchChatHistory` 로직을 주석 처리.
- **데이터 파싱 로직 보정**: 
    - 초기 `res.data` 경로에서 실제 로그 분석을 통해 `res.result.data` 경로를 찾아내어 연동 성공.
    - `userList` 및 `userCount` 필드 추출 시 다양한 명명 규칙(snake_case 등)에 대응하는 유연한 파싱 로직 적용.
- **입장 알림 최적화**: 구독 안착 시간을 확보하기 위해 `ENTER` 메시지 발송 타이밍을 0.5초로 조정.

### 2. 접속자 인지 UI 구현 (`OpenChat.tsx`)
- **실시간 인원수 배지**: 채팅 토글 버튼 및 헤더에 현재 접속자 수를 실시간으로 노출.
- **접속자 명단 뷰(Presence View) 신설**:
    - 헤더의 유저 아이콘 클릭 시 메시지 창이 명단 리스트로 부드럽게 전환(slide-in 애니메이션).
    - 본인 계정 식별 라벨("나") 및 유저별 온라인 상태 표시 추가.
- **반응형 대응**: 명단 보기 모드에서 입력 영역을 숨겨 모바일 환경에서도 리스트 가독성 확보.

- 별도의 새로고침 없이 유저의 입장/퇴장이 명단과 인원수에 즉시 반영됨.
- 히스토리 제거로 더욱 가볍고 실시간 대화에 집중된 채팅 환경 구축.
