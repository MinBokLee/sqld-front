# Detailed Work Summary: 2026-03-31

## 1. 개요 및 세션 목표
본 세션에서는 게시글 작성 페이지의 1단계 모듈화를 수행하고, 시스템 전반에서 발생하던 이미지 경로 오류(401/404) 및 브라우저 히스토리 꼬임 문제를 근본적으로 해결하여 UX와 시스템 안정성을 대폭 강화함.

## 2. 주요 작업 및 기술적 성과

### 2.1. 게시글 작성 페이지(`WritePostPage.tsx`) 모듈화 (1단계)
*   **컴포넌트 추출**: 비즈니스 로직과 분리 가능한 UI 요소를 별도 파일로 독립시킴.
    *   `src/components/ImageLightbox.tsx`: 이미지 확대 보기 로직 분리.
    *   `src/components/WritingGuideAside.tsx`: 우측 작성 가이드 사이드바 분리.
*   **코드 최적화**: 메인 파일의 복잡도를 낮추고 가독성을 향상함.

### 2.2. 이미지 경로 정규화 및 오류 해결 (401/404 대응)
*   **원인 분석**: 서버에서 내려주는 경로(`uploads/...`)와 프론트엔드의 접두어 부착 로직이 충돌하여 중복 경로(`uploads/uploads/...`) 및 슬래시 누락이 발생함.
*   **해결 전략 (Trust the Server)**:
    *   `fixImagePath` 수정: 서버 로컬 물리 경로(`/Users/...`)인 경우에만 파일명을 추출하고, 나머지는 서버가 준 값을 신뢰하여 맨 앞 슬래시(`/`)만 보정함.
    *   `fixContentHtml` 수정: 본문 HTML 내의 중복 패턴을 안전하게 통합하되 슬래시가 누락되지 않도록 정규표현식 강화.
    *   적용 대상: `WritePostPage.tsx`, `ExamDetailPage.tsx`

### 2.3. 내비게이션 및 히스토리 로직 정상화
*   **현상 해결**: 뒤로가기 시 가짜 히스토리가 쌓여 페이지가 튕기거나 [나가기] 버튼이 작동하지 않던 현상을 해결.
*   **로직 정교화**: 
    *   이탈 방지 모달 출력 시 가짜 상태를 단 하나만 유지하도록 관리.
    *   이탈 확정 시 `window.history.go(-2)`를 통해 가짜 상태와 현재 페이지를 한꺼번에 제거하여 정확히 [상세 페이지]로 안착시킴.
    *   글 저장 성공 시 `window.history.back()` 후 `replace: true` 이동을 통해 히스토리 스택을 깨끗하게 정리함.

### 2.4. 사용자 편의성 및 UI 개선
*   **ConfirmModal 강화**: 모달이 열린 상태에서 **엔터키(Enter)**를 누르면 즉시 확인이 실행되고, **ESC키**를 누르면 닫히도록 키보드 이벤트 리스너 추가.
*   **스크랩 메시지 동적화**: 스크랩 등록/취소 시 하드코딩된 문구 대신 서버 응답 메시지(`response.data.message`)를 출력하도록 수정.
